API/WS 列表

  • /api/av/addAttributeViewBlocks - 向指定的属性视图（数据库）中添加块。这些块将成为数据库的行。
  • /api/av/addAttributeViewKey - 向指定的属性视图（数据库）中添加一个新的列（Key）。
  • /api/av/appendAttributeViewDetachedBlocksWithValues - 创建新的游离块（独立块），并将其添加到指定的属性视图中，同时为这些新块设置属性值。
  • /api/av/duplicateAttributeViewBlock - 复制指定的属性视图块（行）。
  • /api/av/getAttributeView - 获取指定 ID 的属性视图（数据库）的结构和配置信息，但不包含其数据行。
  • /api/av/getAttributeViewFilterSort - 获取指定属性视图（数据库）特定视图（View）的过滤（Filter）和排序（Sort）规则。
  • /api/av/getAttributeViewKeys - 获取指定属性视图（数据库）的所有列 (Key) 的信息。
  • /api/av/getAttributeViewKeysByAvID - 获取指定属性视图（数据库）的所有列 (Key) 的信息。根据 Go 源码，此接口与 `/api/av/getAttributeViewKeys` 功能高度相似，可能存在冗余，主要区别在于请求参数字段名不同 (`avID` vs `id`)。
  • /api/av/getAttributeViewPrimaryKeyValues - 获取指定属性视图（数据库）的主键列（通常是文本列）的值，支持分页和关键词搜索。常用于关系列选择关联块。
  • /api/av/getMirrorDatabaseBlocks - 获取指定属性视图（数据库）的所有镜像块（Mirror Blocks）的 ID 列表。
  • /api/av/removeAttributeViewBlocks - 从指定的属性视图（数据库）中移除块（行）。
  • /api/av/removeAttributeViewKey - 从指定的属性视图（数据库）中移除列 (Key)。
  • /api/av/renderAttributeView - 渲染指定的属性视图，返回其结构和数据。支持分页和过滤。
  • /api/av/renderHistoryAttributeView - 根据指定的属性视图 ID 和历史记录时间戳，渲染该属性视图在特定历史时刻的状态和数据。
  • /api/av/renderSnapshotAttributeView - 根据指定的属性视图 ID 和仓库快照 (Snapshot) 索引，渲染该属性视图在特定快照中的状态和数据。
  • /api/av/searchAttributeView - 根据关键词搜索属性视图 (Attribute Views)。可以指定需要排除的属性视图。
  • /api/av/searchAttributeViewNonRelationKey - 在指定的属性视图 (Attribute View) 中，根据关键词搜索非关系类型的列 (Key)。
  • /api/av/searchAttributeViewRelationKey - 在指定的属性视图 (Attribute View) 中，根据关键词搜索关系类型的列 (Key)。关系列用于链接其他属性视图。
  • /api/av/setAttributeViewBlockAttr - 更新属性视图 (Attribute View) 中指定块（行）的某个属性（单元格）的值。
  • /api/av/setDatabaseBlockView - 设置数据库块（属性视图块）的默认显示视图 (View)。当用户打开该属性视图时，将默认展示这里设置的视图。
  • /api/av/sortAttributeViewKey - 对指定属性视图 (Attribute View) 中的列 (Key) 进行排序，改变它们在界面上的显示顺序。
  • /api/av/sortAttributeViewViewKey - 对指定属性视图 (Attribute View) 的特定视图 (View) 中的列 (Key) 进行排序，改变它们在该特定视图下的显示顺序。这允许不同的视图（如表格、列表）拥有独立的列排序。
