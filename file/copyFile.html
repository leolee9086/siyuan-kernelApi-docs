复制文件/目录 API (`/api/file/copyFile`)

在 GitHub 上查看源码 返回文件 API 列表 返回 API 主页

接口描述

该接口用于复制工作空间数据目录 (`data/`) 下的单个文件或目录。

如果目标路径已存在，复制操作将会失败。

请求

方法: POST

路径: `/api/file/copyFile`

认证: 需要 Token

请求体 (JSON)

参数 类型 必需 描述
`fromPath` string 是 源文件或目录的路径，相对于工作空间数据目录 (`data/`)。
`toPath` string 是 目标文件或目录的路径，相对于工作空间数据目录 (`data/`)。

请求示例

复制文件:

{
    "fromPath": "assets/source-image.png",
    "toPath": "assets/copied-image.png"
}

复制目录:

{
    "fromPath": "widgets/my-widget",
    "toPath": "widgets/my-widget-copy"
}

响应

成功响应 (200 OK)

成功复制后，返回一个空数据对象。

{
    "code": 0,
    "msg": "",
    "data": null
}

失败响应

如果请求失败（例如，源文件不存在、目标路径已存在、路径无效、权限不足等），将返回错误信息。

{
    "code": -1,
    "msg": "Source file not found [path=...]", // 或 "Target path exists [path=...]", "Invalid path [...]", "Copy file failed: ..."
    "data": null
}
