删除文件/目录 API (`/api/file/removeFile`)

在 GitHub 上查看源码 返回文件 API 列表 返回 API 主页

接口描述

该接口用于删除工作空间数据目录 (`data/`) 下的单个文件或整个目录（包括其内容）。

注意: 请谨慎使用此接口，删除操作通常是不可逆的（除非系统回收站功能可以恢复）。

请求

方法: POST

路径: `/api/file/removeFile`

认证: 需要 Token

请求体 (JSON)

参数 类型 必需 描述
`path` string 是 要删除的文件或目录的路径，相对于工作空间数据目录 (`data/`)。

请求示例

删除文件:

{
    "path": "assets/image-to-delete.png"
}

删除目录:

{
    "path": "widgets/widget-to-remove"
}

响应

成功响应 (200 OK)

成功删除后，返回一个空数据对象。

{
    "code": 0,
    "msg": "",
    "data": null
}

失败响应

如果请求失败（例如，路径无效、文件或目录不存在、权限不足等），将返回错误信息。

{
    "code": -1,
    "msg": "Remove file failed: remove [...] The system cannot find the file specified.", // 或 "Invalid path [...]"
    "data": null
}
