读取剪贴板文件路径

在 GitHub 上查看源码

POST /api/clipboard/readFilePaths

描述

尝试从系统剪贴板中读取文件路径列表。这通常用于处理用户从文件管理器复制文件后，在思源笔记中进行粘贴操作的场景。

注意： 此功能在 Linux 系统上可能受限。

请求参数

此接口不需要请求参数。

响应结果

响应体是 JSON 格式。code 为 0 表示成功。data 是一个包含从剪贴板读取到的文件绝对路径的字符串数组。如果剪贴板中没有文件路径，则返回空数组 []。


{
  "code": 0,
  "msg": "",
  "data": [
    "/path/to/file1.txt",
    "/path/to/image.png"
    // ... more file paths
  ]
}
