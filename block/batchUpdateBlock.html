批量更新块 (batchUpdateBlock)

首页 | Block API | GitHub Source

功能描述

此 API 端点 POST /api/block/batchUpdateBlock 用于在一个原子事务中批量更新多个块的内容。

这对于需要同时修改多个块并确保它们要么全部成功、要么全部失败的场景非常有用。

API 会将所有指定的更新操作打包成一个事务执行，执行后会强制刷新数据库队列，并通过 WebSocket 广播变更。

注意： 更新文档块（`NodeDocument`）时，此 API 会先删除文档内所有现有块，然后将新内容追加进去，行为与更新普通块不同。

请求参数

请求体必须是 JSON 格式，包含以下参数：

  • blocks (array): 必填。一个包含多个块更新信息的对象数组。
    • 数组中的每个对象代表一个要更新的块，包含以下字段：
      • id (string): 必填。要更新的块的 ID。
      • data (string): 必填。块的新内容。
      • dataType (string): 必填。`data` 的格式，必须是 `"markdown"` 或 `"dom"`。

请求示例:


{
  "blocks": [
    {
      "id": "20230315180000-abcdefg",
      "data": "这是块 1 的新 Markdown 内容",
      "dataType": "markdown"
    },
    {
      "id": "20230315180100-hijklmn",
      "data": "<div data-type=\"NodeParagraph\" class=\"p\"><div contenteditable=\"true\">这是块 2 的新 DOM 内容</div></div>",
      "dataType": "dom"
    }
    // ... 可以有更多要更新的块
  ]
}

响应结果

成功的请求将返回一个 JSON 对象，code 为 0，data 字段包含一个事务数组（通常只包含一个事务，包含了所有执行的操作）。

返回的事务结构与 /api/transactions 类似，包含了 doOperations 和 undoOperations。


{
  "code": 0,
  "msg": "",
  "data": [
    {
      "doOperations": [
        {
          "action": "update", // 或者 delete, appendInsert 等
          "id": "20230315180000-abcdefg",
          "data": "<div data-node-id=\"20230315180000-abcdefg\" ...>...</div>" // 处理后的 DOM
        },
        {
          "action": "update",
          "id": "20230315180100-hijklmn",
          "data": "<div data-node-id=\"20230315180100-hijklmn\" ...>...</div>"
        }
        // ... 所有操作
      ],
      "undoOperations": [
        // ... 对应的撤销操作
      ],
      "timestamp": 1678886400000 // 实际时间戳
    }
  ]
}

如果操作失败（例如参数错误、块不存在、数据解析失败等），将返回非 0 的 code 和具体的错误信息 msg。


{
  "code": -1,
  "msg": "parse tree failed", // 示例错误信息
  "data": null
}
