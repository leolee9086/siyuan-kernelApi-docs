设置上传错误日志 (setUploadErrLog)

首页 | System API | GitHub Source

功能描述

此 API 端点 POST /api/system/setUploadErrLog 用于设置是否允许思源笔记在发生错误时自动上传错误日志。

它修改配置文件中的 conf.System.UploadErrLog 项。设置成功后会向用户发送通知，并延迟3秒返回结果。

上传错误日志有助于开发者诊断和修复问题。

请求参数

请求体必须是 JSON 格式，包含以下参数：

  • uploadErrLog (boolean): 必填。是否允许上传错误日志。
    • true: 允许上传错误日志。
    • false: 禁止上传错误日志。

请求示例 (允许上传):


{
  "uploadErrLog": true
}

请求示例 (禁止上传):


{
  "uploadErrLog": false
}

响应结果

成功的请求将返回一个 JSON 对象，code 为 0，data 为 null。同时会在界面上显示一条提示消息。

注意：此 API 会延迟 3 秒钟才返回响应，以便用户看到提示消息。


{
  "code": 0,
  "msg": "",
  "data": null
}

如果操作失败（例如参数无效），将返回非 0 的 code 和具体的错误信息 msg。


{
  "code": -1, // 示例错误码
  "msg": "Invalid parameter", // 示例错误信息
  "data": null
}
