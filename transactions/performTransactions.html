执行事务 (performTransactions)

首页 | Transactions API | GitHub Source

功能描述

此 API 端点 POST /api/transactions 是思源笔记执行各种数据修改操作的核心入口。

它接收一个包含多个具体操作（称为 "事务" 或 Transaction）的列表，然后在后端按顺序执行这些操作。常见的操作包括插入块、更新块、移动块/文件、删除块/文件等。

执行完成后，API 会返回处理后的事务列表（可能包含每个操作的结果或错误信息），并通过 WebSocket 将这些事务推送给其他客户端，以保持数据同步。

这是思源笔记中非常底层的核心 API，用于实现绝大部分的数据修改功能。

请求参数

请求体必须是 JSON 格式，包含以下参数：

  • transactions (array): 必填。一个事务对象 (Transaction) 的数组，每个对象代表一个要执行的操作。
    • 每个事务对象通常包含 doOperations 数组，其中每个元素代表一个具体的操作步骤 (Operation)。
    • 每个操作步骤包含 action (操作类型，如 "updateBlock", "moveDocsTo" 等) 和相应的参数 (如 id, data, parentID 等)。
    • action 的具体类型和所需参数非常多，取决于具体的操作。建议通过开发者工具观察前端发送的请求来了解特定操作的结构。
  • reqId (number): 必填。一个时间戳（通常是 `Date.now()`），用于标识本次请求，并会被设置到每个事务对象上。
  • app (string): 必填。发起请求的应用实例 ID，通常是一个 UUID。
  • session (string): 必填。发起请求的会话 ID（WebSocket 连接 ID），通常是一个 UUID。

请求示例 (简化结构，仅作示意):


{
  "app": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "session": "yyyyyyyy-yyyy-yyyy-yyyy-yyyyyyyyyyyy",
  "reqId": 1678886400000,
  "transactions": [
    {
      "doOperations": [
        {
          "action": "updateBlock",
          "id": "20230315180000-abcdefg",
          "data": "# 这是新的标题\n这是更新后的内容"
        }
      ]
    },
    {
      "doOperations": [
        {
          "action": "moveBlock",
          "id": "20230315180100-hijklmn",
          "previousID": "",
          "parentID": "20230315175900-opqrstu"
        }
      ]
    }
    // ... 可能有更多事务
  ]
}

响应结果

成功的请求将返回一个 JSON 对象，code 为 0，data 字段包含处理后的事务列表。

处理后的事务对象中可能包含 undoOperations（用于撤销操作）以及 retData（某些操作的返回值）。


{
  "code": 0,
  "msg": "",
  "data": [
    {
      "doOperations": [
        {
          "action": "updateBlock",
          "id": "20230315180000-abcdefg",
          "data": "# 这是新的标题\n这是更新后的内容"
          // 可能有 retData
        }
      ],
      "undoOperations": [
        {
          "action": "updateBlock",
          "id": "20230315180000-abcdefg",
          "data": "# 旧标题\n旧内容"
        }
      ],
      "timestamp": 1678886400000
    },
    {
      "doOperations": [
        {
          "action": "moveBlock",
          "id": "20230315180100-hijklmn",
          "previousID": "",
          "parentID": "20230315175900-opqrstu"
        }
      ],
      "undoOperations": [
         // ... 对应的撤销操作
      ],
      "timestamp": 1678886400000
    }
    // ...
  ]
}

如果操作失败（例如请求解析错误、系统未启动完成等），将返回非 0 的 code 和具体的错误信息 msg。


{
  "code": -1,
  "msg": "Booting, please wait... [Progress: 85%]",
  "data": {
    "closeTimeout": 5000
  }
}
